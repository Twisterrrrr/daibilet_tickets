# Открытые вопросы — решения (19.02.2026)

> Исключены: деплой, оплата YooKassa (отдельные задачи).

---

## 1. Переклассификация уже загруженных событий

**Решение:** Массовый скрипт переклассификации + лог изменений, а не повторный sync.

**Почему:**
- Повторный sync не исправит логические ошибки в правилах.
- EventOverride не масштабируется для сотен кейсов.
- Классификация — SEO и фильтрация → стратегический слой.

**Как:**
- Скрипт `reclassify-events.ts`
- dry-run режим (лог diff: old → new)
- Отчёт: сколько событий сменили subcategory
- После — выборочная ручная проверка 50 случайных

**Приоритет:** высокий.

---

## 2. EventOverride.subcategories

**Решение:** Сделать override жёстким.

**Нужно:**
- allow override category
- allow override subcategory
- allow clear subcategory (`override.subcategory = null` → принудительно убрать)
- Не только tagsAdd/tagsRemove — админ должен полностью контролировать каталог

**Приоритет:** средний, но важен для чистоты данных.

## 3. GiftCertificate — email получателю

**Решение:** Это критический UX-блокер. Реализовать обязательно.

**Нужно:**
- Шаблон письма (HTML)
- Отправка в fulfillment после COMPLETED
- Retry логика (через очередь)
- Лог в Admin → статус «email_sent»

Без этого сертификат — полуфункция.

**Приоритет:** высокий.

## 4. Скрыть корзину

**Решение:** Да. Скрыть корзину.

**Почему:**
- TC widget, redirect, API checkout, split YooKassa — единая корзина создаёт иллюзию единого платежа
- Снижает когнитивную нагрузку и путаницу

**Как:**
- Скрыть корзину
- Оставить прямой checkout per-offer
- Вернуть корзину после унификации платежей

**Приоритет:** высокий.

**Решение:** Аудит соответствия спецификации.

## 7. Унификация session.prices

**Решение:** Да. Обязательно.

**Почему:**
- TC: price; Teplohod: amount/price — разные форматы
- Снижает баги типа «1 ₽»

**Как:**
- Backend: нормализатор `NormalizedPrice { amountCents, currency, available }`
- Фронт работает только с ним

**Приоритет:** высокий.

**Приоритет:** высокий.
