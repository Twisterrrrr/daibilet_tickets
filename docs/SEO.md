# 1. Архитектура SEO

## 1.1 Принцип

SEO строится по модели:

-   Города → Категории → Фильтры → Карточки
-   Один URL = один поисковый интент
-   Нет дублей, нет мусорных комбинаций
-   Качество \> количество

------------------------------------------------------------------------

# 2. URL-структура (текущая)

-   /
-   /events
-   /events/\[slug\]
-   /venues
-   /venues/\[slug\]
-   /cities
-   /cities/\[slug\]
-   /cities/\[slug\]/museums
-   /blog
-   /blog/\[slug\]

Будущая цель:

/{city}/{type} /{city}/{type}/{filter}

где: type = excursions \| museums \| events

------------------------------------------------------------------------

# 3. Sitemap

-   /sitemap.xml
    -   sitemap-static
    -   sitemap-cities-catalog
    -   sitemap-cities-filters
    -   sitemap-offers
    -   sitemap-articles

В sitemap попадают только: - 200 OK - robots != noindex - filters count
≥ 6 - реальный lastmod (updatedAt)

# 4. SEO-автогенерация

## 4.1 Модели

### SeoTemplate

-   type: excursion \| venue \| event
-   level: 0 \| 1 \| 2
-   template (с переменными)

### SeoContent

-   cityId
-   type
-   filtersKey
-   title
-   h1
-   description
-   body

## 4.2 Уровни

L0 --- всегда\
L1 --- count ≥ 6\
L2 --- count ≥ 10 + whitelist

## 4.3 Переменные шаблонов

{{city}}, {{city_gen}}, {{type_title}}, {{filter_title}}, {{count}},\
{{price_from}}, {{price_to}}, {{season}}, {{year}}

------------------------------------------------------------------------

# 5. Anti-thin-content

Индексация разрешена только если:

-   offersCount ≥ 6
-   body ≥ 600 символов
-   description 120--160
-   title ≠ h1
-   ≥ 2 операторов
-   средняя цена \> 0

Если не проходит: robots = noindex,follow

------------------------------------------------------------------------

# 6. Landing-модель

Draft → Published

Публикация только при Quality Score ≥ 4

Quality Score:

+2 eventCount ≥ 10\
+1 uniqueOperators ≥ 3\
+2 hasSeoText\
+1 hasFaq\
+1 avgRating ≥ 4.3

------------------------------------------------------------------------

# 7. Canonical и каннибализация

-   Один slug на один интент
-   Без синонимов
-   canonical указывает на себя
-   родительская и дочерняя страницы имеют разные meta-title

------------------------------------------------------------------------

# 8. SeoMeta

Entity types: CITY, EVENT, VENUE, LANDING, ARTICLE, COMBO

Приоритет: 1. Manual 2. Auto 3. Default

Поля: title, description, h1, canonicalUrl, robots, ogTitle,
ogDescription, ogImage, jsonLd

------------------------------------------------------------------------

# 9. JSON-LD

Event: - name - startDate - offers - location - organizer

Museum / Venue: - address - geo - openingHours

Article: - headline - author - datePublished

------------------------------------------------------------------------

# 10. Стратегия масштабирования

Через: - города - категории - статьи - перелинковку

Не через: - перебор фильтров - дубли - автогенерацию мусора

------------------------------------------------------------------------

# 11. Перелинковка

Каждая страница должна иметь: - блок "Читайте также" - ссылки на
родительскую категорию - соседние фильтры - топ-карточки - 1--2 статьи

------------------------------------------------------------------------

# 12. Production-шаблоны

Файл: prisma/seed-seo-templates.ts

Правила: - 600--1200 символов - упоминание count - ценовой диапазон -
блок "как выбрать" - подтверждение билета

------------------------------------------------------------------------

# 13. Правило 10×10

Для каждого города: - 1 главная страница - 3 категории - 3--5 фильтров -
3--5 статей

Итого: 10--15 сильных URL на город

------------------------------------------------------------------------

# 14. Главный принцип

10 сильных страниц \> 300 пустых.
