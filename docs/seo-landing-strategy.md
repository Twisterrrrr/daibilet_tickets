# Стратегия авто-лендингов: качество вместо мусора

> Последнее обновление: 2026-02-21

## 1. Главный принцип

**Автоматически можно создавать только кандидатов, а не страницы.**

| Сущность | Описание |
|----------|----------|
| **LandingDraft** | Кандидат. Создаётся автоимпортом. |
| **LandingPage** | Публичная страница. Публикация только по порогам качества. |

Автоимпорт → создаёт Draft.  
Публикация → только после прохождения фильтра.

---

## 2. SEO-порог публикации

Перед тем как страница станет PUBLISHED:

| Критерий | Порог |
|----------|-------|
| Количество событий | ≥ 6–8 |
| Минимум 2 разных операторов | да |
| Средняя цена > 0 | да |
| Есть рейтинг ≥ 4.0 | желательно |
| Уникальный SEO-текст | ≥ 400 символов |

Не проходит → DRAFT + noindex.

---

## 3. Разрешённые типы автокандидатов

**Неправильно** — перебор всех комбинаций:
```
CITY × CATEGORY × SUBCATEGORY × TAG × SEASON × AUDIENCE
```
Это сотни страниц на один город.

**Правильно:**
- `CITY + CATEGORY`
- `CITY + SUBCATEGORY` — только если событий ≥ X (например 8)
- `CITY + TAG` — только если tag.frequency ≥ Y

Всё остальное — вручную.

---

## 4. Порог частоты (до черновика)

Перед созданием draft:

```sql
SELECT COUNT(*) FROM events
WHERE city = X AND subcategory = Y
```

Если `< 5` → **не создаём даже черновик**.

Убирает ~80% мусора.

---

## 5. Один URL — одна поисковая интент-модель

Не делать:
- `/spb/night-excursions`
- `/spb/excursions-at-night`
- `/spb/excursions-night`

Один канонический slug:
```
/cities/{city}/excursions/{filter}
/cities/{city}/museums/{filter}
/cities/{city}/events/{filter}
```

Slug — по фиксированной схеме, без синонимов.

---

## 6. Все автосозданные лендинги = noindex

Пока:
- нет SEO-текста
- нет FAQ
- нет внутренней перелинковки

В мета:
- `robots = "noindex,follow"`
- `autoGenerated = true`

Только после **ручной активации**:
- `robots = "index,follow"`
- `autoGenerated = false`
- `status = PUBLISHED`

---

## 7. Landing Quality Score

В админке:

```
qualityScore =
  + (eventCount >= 10 ? 2 : 0)
  + (uniqueOperators >= 3 ? 1 : 0)
  + (hasSeoText ? 2 : 0)
  + (hasFaq ? 1 : 0)
  + (avgRating >= 4.3 ? 1 : 0)
```

Публикация разрешена только если **score ≥ 4**.

---

## 8. Не индексировать пустые фильтры

| Условие | Действие |
|---------|----------|
| 0 событий | 410 или 404 |
| < 3 событий | noindex |
| Только 1 событие | редирект на карточку события |

---

## 9. Контроль каннибализации

- canonical у дочерней страницы (walking) → self
- у родительской (excursions) — уникальные meta-title/h1, не дублирующие дочерние
- Meta-title не должен быть «Экскурсии в СПб» для всех страниц подряд

---

## 10. Простая формула перед созданием landing

```ts
if (eventCount < 6) return;
if (uniqueOperators < 2) return;
if (avgRating < 4.0) return;
// если все 3 выполнены → создать DRAFT
```

---

## 11. Когда авто-лендинги реально полезны

- база > 500 событий в городе
- есть явные кластеры
- есть стабильный поисковый спрос

Пока база маленькая — лучше вручную выбирать 5–10 сильных кластеров.

---

## 12. Рекомендация для проекта

С учётом стратегии из `Project.md`:

Делать авто-draft:
- только **CITY + CATEGORY**
- только **CITY + SUBCATEGORY** (если ≥ 8 событий)

Всё остальное — вручную.

---

## 13. Главное правило

> **SEO масштабируется не количеством URL, а количеством качественных кластеров.**
>
> 10 сильных страниц > 300 пустых.
