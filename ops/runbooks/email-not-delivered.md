# Runbook: Письмо не доставлено клиенту

## Симптом

Клиент не получил email (подтверждение заказа, ваучер, сброс пароля и т.п.).

## Проверки

1. **Спам:** клиенту проверить папку «Спам» и «Промоакции»
2. **Логи:** MailService / BullMQ emails queue — job выполнен? Ошибка SMTP?
3. **Админка → Orders:** письмо должно было отправиться при переходе в PAID/FULFILLED
4. **Idempotency / дубли:** не заблокирована ли отправка из‑за дубля (EMAIL_SEND scope)?

## Диагностика

- Ошибки SMTP: 550, 554 — домен/адрес заблокирован
- Timeout — SMTP-сервер не отвечает
- BullMQ: job в failed с `reason` — смотреть stack

## Действия в админке

1. **Resend email:** Admin → Order → «Отправить письмо повторно» (если есть)
2. **Проверить SMTP:** `MAIL_HOST`, `MAIL_USER`, env — корректны ли?
3. **Retry failed job:** Admin → Jobs → Failed → retry для `emails` queue

## Эскалация

- SMTP постоянно возвращает ошибку
- Нужно сменить отправителя / домен
- Клиент уверяет, что письма не приходят, а в логах «sent»
